Maps and sets Maps Map What is a Map: A Map is a collection of keyed values where each key may occur only once. Unlike regular objects in JS, which use strings and symbols as keys, Maps accept any type of key. Creating a Map: let map = new Map(); Adding Key-Value Pairs: map.set('key1', 'value1'); map.set(2, 'value2'); map.set(true, 'value3'); Retrieving a Value: console.log(map.get('key1')); // Output: value1 console.log(map.get(2)); // Output: value2 Checking for Key Existence: console.log(map.has('key1')); // Output: true console.log(map.has('key3')); // Output: false Deleting an Entry: map.delete('key1'); // Deletes the key-value pair with key 'key1' Removing all Entries: map.clear(); // Removes all key-value pairs from the map Iterating over a Map: map.set('a', 1); map.set('b', 2); for (let [key, value] of map.entries()) { console.log(key, value); } // Output: // a 1 // b 2 Set What is a Set: A Set is a collection of unique values where each value may occur only once. It's particularly useful when you need to ensure that a collection contains no duplicate elements. Creating a Set: let set = new Set(); Adding a Value: set.add(1); set.add(2); set.add(1); // Ignored, as it already exists Removing a Value: set.delete(1); // Removes the value 1 from the set Checking for Value Existence: console.log(set.has(2)); // Output: true console.log(set.has(3)); // Output: false Removing all Values: set.clear(); // Removes all values from the set Iterating over a Set: set.add('a'); set.add('b'); set.forEach(value => { console.log(value); }); // Output: // a // b Importance of Map and Set Map: Key-Value Pairing: Maps allow for the association of keys with values, offering a more versatile alternative to object properties. Order Preservation: They maintain the order of insertion, which is crucial for scenarios where the sequence of elements matters. Efficient Searching: Maps provide efficient key-based retrieval, outperforming object iteration in scenarios requiring frequent access to values. Set: Uniqueness: Sets ensure that each value appears only once, making them indispensable for scenarios where duplicate values are undesirable. Efficient Searching: Sets offer efficient methods for checking the existence of values within the collection, facilitating fast lookups. Order Preservation: Similar to Maps, Sets also maintain the order of insertion, which is beneficial when preserving sequence matters. Both Map and Set data structures enhance the capabilities of collections, providing efficient manipulation and retrieval methods. They offer powerful tools for managing data, ensuring uniqueness, and preserving order, thereby contributing to cleaner, more maintainable code.
